> 现在你支持的agent team怎么使用？

● 我可以通过 Task 工具启动专门的子代理来处理复杂任务。以下是目前可用的代理类型：

  - Bash — 执行 shell 命令，适合 git 操作、命令行任务
  - general-purpose — 通用代理，适合研究复杂问题、搜索代码、执行多步骤任务
  - Explore — 快速探索代码库，按模式查找文件、搜索关键词、回答代码库相关问题
  - Plan — 软件架构代理，设计实现方案、识别关键文件、考虑架构权衡
  - django-fullstack-developer — Django 全栈开发专家，处理 Django 后端和前端集成
  - qa-web-tester — Web 应用质量保证测试，验证功能、生成测试报告
  - claude-code-guide — 回答关于 Claude Code CLI 工具、Agent SDK、API 使用的问题

  使用方式很简单 — 你不需要手动调用它们。当你给我一个任务时，我会根据需要自动选择合适的代理。比如：

  - 需要深入探索一个不熟悉的代码库 → 我会用 Explore 代理
  - 需要规划一个复杂功能的实现 → 我会用 Plan 代理
  - 多个独立的搜索/研究任务 → 我会并行启动多个代理提高效率

  你只需要正常描述你的需求就行，代理调度是我在幕后处理的。有具体的任务想试试吗？
